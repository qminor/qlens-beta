1. Purge all modules if any are currently loaded ('module purge'). Install conda/miniconda with wget if not already installed. Next...

conda create --name <environment_name> python=3.13 openmpi pybind11 cxx-compiler gfortran cfitsio fftw mkl mkl-include numpy matplotlib libumfpack armadillo

*NOTE*: if it complains, you might have to add '-c conda-forge' to the above, namely

conda create --name <environment_name> python=3.13 -c conda-forge openmpi pybind11 cxx-compiler gfortran cfitsio fftw mkl mkl-include numpy matplotlib libumfpack armadillo

2. Now activate your environment ('conda activate <environment_name>').

3. Next, go to your .../qlens-beta/contrib dir, and do:

git clone https://github.com/JohannesBuchner/MultiNest.git
cd MultiNest/build
cmake ..
make

4. Now go back to .../qlens-beta/contrib dir, and do:

git clone https://github.com/PolyChord/PolyChordLite.git
cd PolyChordLite
make

5. Now edit the Makefile_conda in qlens-beta. For the lines HOMEDIR and ENV_NAME, substitute your home directory and conda environment name. For CONDA_DIR and QLENS_DIR, make sure it matches your conda directory and qlens directories, respectively.



6. Now go back to the terminal. You can either rename/copy Makefile_conda to Makefile (in which case you'll just run "make"), or you can just run "make -f Makefile_conda". Either way, run make and watch magic happen. If something goes wrong, yell at Quinn for forgetting to include something important in this file (or maybe just send him the error message and ask what's up).

7. Before trying to run qlens, make sure your PYTHONPATH has the folder where the qlens python library is, something like e.g. /mendel-nas1/<username>/qlens-beta/python/qlens.

8. If all has gone well, cd into the qlens-beta/python folder and try running 'python3 alphafit.py'. If that script works, now try 'python3 -i delaunay_fit_demo.py'. If that works, then you're all set!
